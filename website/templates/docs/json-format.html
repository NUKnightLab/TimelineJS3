TimelineJS 3 JSON data format

Every timeline needs data about the events it is to show. Most people are happy to just use a Google spreadsheet to configure their Timeline, but if you want to write code to dynamically create or update your timeline, you'll need to understand how to format the data.

If you just want to dive in, you can probably copy from one of our examples (like the one about Whitney Houston). Otherwise, complete documentation is below.

After you have worked out how to create JSON data for a Timeline, you'll also need to put it on the page.

The TimelineJS 3 JSON format consists of a single JSON object with the following properties:

events (required)
value: a JSON list of "slide" objects (see below)
title (optional)
value: a "slide" object (see below)
scale (optional)
value: either human or cosmological. If no scale is specified, the default is human. The cosmological scale is required to handle dates in the very distant past or future. (Before Tuesday, April 20th, 271,821 BCE after Saturday, September 13 275,760 CE) For the cosmological scale, only the year is considered, but it's OK to have a cosmological timeline with years between 271,821 BCE and 275,760 CE.

Slide objects are JSON objects with the following properties:

start_date (required, except for 'title' slides)
value: a "date" object (see below)
end_date (optional)
value: a "date" object (see below)
text (optional but recommended)
value: a "text" object (see below)
media (optional)
value: a "media" object (see below)
group (optional)
value: any text. If present, Timeline will organize events with the same value for group to be in the same row or adjacent rows, separate from events in other groups. The common value for the group will be shown as a label at the left edge of the navigation.
display_date (optional)
value: a string which will be used when Timeline displays the date for this. If used, override's display_date values set on the start or end date for this event, which is useful if you want to control how the two dates relate to each other.
background (optional)
value: a javascript object. The object can have these properties:
  • url: the fully-qualified URL pointing to an image which will be used as the background
  • color: a CSS color, in hexadecimal (e.g. #0f9bd1) or a valid CSS color keyword.

Date objects are JSON objects with the following properties:

year (required)
value: a number. Don't use commas. BCE years should be negative numbers. Don't use the letters "BC", "BCE" or any others.
month (optional)
value: a number from 1-12 (Javascript experts don't outsmart yourselves: Timeline corrects for Javascript's strange use of "0" for "January", etc.)
day (optional)
value: a number
hour (optional)
value: a number from 0-23.
minute (optional)
value: a number from 0-59
second (optional)
value: a number from 0-59
millisecond (optional)
value: a number
display_date (optional)
value: a string for presenting the date. Useful if Timeline's date formatting doesn't fit your needs.

Text objects are JSON objects with the following properties. For each slide, text objects are optional.

headline (required)
value: any text. HTML markup is OK. Blank is also OK, but the key must be present in the JSON object.
text (optional)
value: any text. HTML markup is OK. Blank is also OK, but the key must be present in the JSON object.

Media objects are JSON objects with the following properties. For each slide, media objects are optional.

url (required)
value: in most cases, a URL (see media type documentation for complete details).
caption (optional)
value: any text. HTML markup is OK.
credit (optional)
value: any text. HTML markup is OK.
